# Fitness Tracker API с PostgreSQL

Проект реализует REST API фитнес-трекера с использованием PostgreSQL в качестве основного хранилища данных.

## Схема базы данных

Система построена на четырех реляционных таблицах:
- `users` — учетные записи пользователей с уникальными логином и email
- `exercises` — каталог упражнений с категориями, сложностью и автором
- `workouts` — записи о тренировках пользователей с временными метками
- `workout_exercises` — ассоциативная таблица, связывающая тренировки и упражнения, хранит параметры выполнения (подходы, повторения, вес, длительность)

Связи реализованы через внешние ключи с каскадным удалением. Для обеспечения целостности добавлены ограничения `NOT NULL`, `UNIQUE` и `CHECK`. Подробная DDL-структура описана в `schema.sql`.

## Запуск проекта

### Требования
- Docker и Docker Compose
- PostgreSQL 16+ (разворачивается автоматически)

### Инструкция
1. Запустите контейнеры:
   ```bash
   docker-compose up --build -d
   ```
2. Дождитесь полного старта. БД автоматически инициализируется скриптами `schema.sql` и `data.sql`.
3. Проверьте логи сервиса:
   ```bash
   docker-compose logs -f api
   ```
4. API доступен по адресу: `http://localhost:8080`

### Параметры подключения к БД
- Хост: `postgres`
- Порт: `5432`
- База: `fitness_db`
- Пользователь: `fitness_user`
- Пароль: `fitness_pass`

## API Операции

Реализованы следующие методы:
- `POST /users` — создание нового пользователя
- `GET /users?login={login}` — поиск пользователя по логину
- `GET /users?name_mask={pattern}` — поиск пользователя по маске имени и фамилии
- `POST /exercises` — создание упражнения
- `GET /exercises` — получение списка упражнений
- `POST /workouts` — создание тренировки
- `POST /workouts/{id}/exercises` — добавление упражнения в тренировку
- `GET /users/{id}/workouts` — получение истории тренировок пользователя
- `GET /users/{id}/workouts/stats` — получение статистики тренировок за период

Готовые SQL-запросы для каждой операции находятся в `queries.sql`. Анализ производительности и оптимизации документирован в `optimization.md`.

## Структура файлов
- `schema.sql` — создание таблиц, индексов и ограничений
- `data.sql` — заполнение тестовыми данными (10+ записей на таблицу)
- `queries.sql` — SQL-запросы для всех API операций
- `optimization.md` — EXPLAIN-планы, обоснование индексов, стратегия партиционирования
- `docker-compose.yaml` — оркестрация сервисов API и PostgreSQL
- `Dockerfile` — сборка бинарного файла сервиса
