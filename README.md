
## Описание схемы БД

Схема включает следующие сущности:

1. **users**
   - Хранит информацию о зарегистрированных покупателях (логин, имя, фамилия, контакты).
   - Первичный ключ: `id`.
   - Ограничения: `UNIQUE` на `login`, `NOT NULL` на основные поля.

2. **products**
   - Содержит каталог товаров (название, цена, описание, остаток на складе, категория).
   - Первичный ключ: `id`.
   - Ограничения: `CHECK` на цену и остаток (не могут быть отрицательными).

3. **cart_items**
   - Связывает пользователей с товарами (список покупок конкретного пользователя).
   - Первичный ключ: `id`.
   - Внешние ключи: `user_id` (ссылка на `users`), `product_id` (ссылка на `products`).
   - Связь: "Один ко многим" (Один пользователь может иметь много товаров в корзине).

**Схема связей:**
`users (1) ──< cart_items (N) >── (1) products`

---


### Запуск

1. Откройте терминал в папке проекта.

2. Запустите контейнеры в фоновом режиме:
   ```bash
   docker-compose up -d
   ```
   
3. Убедитесь, что база данных запущена:
   ```bash
   docker-compose ps
   ```
   *Статус контейнера должен быть `Up (healthy)`.*

### Подключение к базе данных

Чтобы войти в консоль управления PostgreSQL, выполните:
```bash
docker-compose exec db psql -U user -d ozon_store
```

Для выхода из консоли введите:
```sql
\q
```
